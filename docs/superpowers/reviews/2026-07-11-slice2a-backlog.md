## Бюджет: привязка и агрегаты

- **[Important]** `apps/server/src/budget/binding.ts` (`bindingOps`/`selectEnvelope`) —
  **write-skew привязки при конкурентных «create транзакции ∥ create конверта»**: селектор
  §2.3 одной tx не видит незакоммиченный конверт другой — транзакция остаётся Unbudgeted,
  хотя конверт «уже есть». Advisory-lock сериализует только уникальность конвертов
  (`assertEnvelopeUnique`), не привязку. Паттерн тот же, что write-skew blocks (ревью
  2026-07-09). Не чинили: требует lock на владельца вокруг всего бюджет-хука; окно —
  миллисекунды, самовосстанавливается любым ребиндингом (правка конверта/транзакции).
- **[Important]** `apps/server/src/executor/executor.ts` (`budgetFollowUpDescs`, условие (а)) +
  `binding.ts:bindingOps` — **detach `orbis/financial` оставляет висящую parent-связь**:
  условие (а) требует `afterAspects['orbis/financial'] !== undefined`, а `bindingOps` при
  отсутствии financial возвращает `[]` — снять устаревшую привязку некому. `children_of`
  конверта покажет не-financial ребёнка; на spent не влияет (SQL-агрегаты фильтруют по
  financial + с фикса финального ревью — по не-шаблонности). Не чинили: зеркальный кейс
  «стал шаблоном» закрыт fix-раундом; detach-ветка требует симметричного unbind-пути —
  закрыть вместе с ним (кандидат — фаза B, вместе с ревизией условия (а)).
- **[Important]** `apps/server/src/budget/aggregates.ts` (`rolloverCreate`, пречек) —
  **TOCTOU NULL-currency-преемника**: пречек с coalesce-валютой идёт вне tx executor'а;
  NULL-currency-конверт, созданный между пречеком и batch, инвариант §2.1 «точная
  комбинация» не поймает. Корень — буква §2.1. **Спек-решение (принять до B2):
  нормализация currency NULL→defaultCurrency при create конверта** — тогда комбинации
  становятся каноничными и щель закрывается. Вписано обязательством в Task B2 плана.
- **[Minor]** `apps/server/src/budget/binding.ts` (`rebindForEnvelope`) — **N+1
  `selectEnvelope`**: ребиндинг зовёт селектор на каждую затронутую транзакцию. На ручных
  объёмах незаметно; при массовом импорте CSV (сотни строк одним batch) станет квадратом.
  Вписано обязательством в Task C1/C2 плана («устранить до массового импорта»).
- **[Minor]** `packages/shared/src/schemas/aspects.ts` (`category_ref: z.string().uuid()`) —
  **конверт-сирота на несуществующий categoryId**: валидация только формата uuid, ссылочной
  проверки категории нет — rollover/`entity_create` создаёт конверт с висящей категорией
  (карточка без имени/иконки). Не чинили: граф намеренно без FK-семантики для aspect-ссылок
  (§2.1); UI выбирает категорию из списка — мусор достижим только через API.

## Recurring / материализация

- **[Minor]** `apps/server/src/recurring/materialize.ts` (`materializationWindow`) —
  **over-approximation окна**: union дизъюнктных date-диапазонов запроса берётся выпуклой
  оболочкой — редкий запрос «янв ∪ дек» материализует год инстансов одним batch, лимитов
  batch нет (cap на размер batch — кандидат). Не чинили: реальные потребители окна (агенда,
  бюджет) задают компактные диапазоны; cap — механическая правка при первом инциденте.
- **[Minor]** `apps/server/src/recurring/materialize.ts` (`materializationWindow`) —
  **окно не знает абсолютных диапазонов дат**: распознаются только относительные токены;
  абсолютный диапазон `occurred_on=2026-06-01..2026-06-30` (грамматика расширяется в B5)
  окно не расширит — инстансы месяца могут быть не материализованы к моменту запроса.
  Вписано обязательством в Task B5 плана («вместе с грамматикой»).
- **[Minor]** `apps/server/src/recurring/post-due.ts` — **postDue для нестандартной TZ без
  теста**: таймзонный путь протестирован только для дефолта (Europe/Moscow); нестандартная
  зона пользователя/мусор в user_settings.timezone — код деградирует до дефолта, но
  pin-теста нет. Тестовый долг, не дефект.

## Журнал / системные строки

- **[Minor]** `apps/server/src/chat/messages.ts` (`excludeInfraSystemRows`) +
  `apps/server/src/executor/undo.ts:64` — **предикаты «системности» с разной семантикой**:
  после унификации фрагмента (fix финального ревью: chat.listMessages и LLM-контекст делят
  один SQL) остаётся undo-путь с формой `->'actions'->0->>'source'` (первый action).
  Эквивалентность форм держится на инварианте «один action на сообщение журнала», который
  нигде не запинен тестом. Кандидат: pin-тест инварианта + свести undo к тому же фрагменту.
- **[Minor]** журнал §7.8 (`chat_messages` глобального треда) — **вечное накопление
  system-audit**: каждый пересчёт агенды/бюджета пишет batch-audit материализации; строки
  скрыты из выдачи и LLM-контекста (SQL-фильтр), но таблица растёт неограниченно. Не
  чинили: append-only журнал — спек-требование (§4.6); нужна политика retention/компакции
  (отдельное спек-решение, не код фазы A).
- **[Minor]** `db/migrations` (GRANT DELETE на `chat_messages`) ↔ уборка processing-маркера
  (`ai/send-message.ts`) — **конфликт с бэклог-пунктом append-only enforcement** (ревью
  2026-07-09): если БД начнёт запрещать DELETE, уборка маркера сломается. Зависимость
  зафиксирована; решать вместе с enforcement'ом (маркер — не журнальная строка, кандидат
  на отдельную таблицу/условие политики).
- **[Minor]** `apps/server/src/ai/send-message.ts` (`PROCESSING_TTL_MS`) — **зомби-цикл
  длиннее TTL снимет чужой маркер**: прогон, живущий дольше 10 минут, считается умершим —
  конкурентный ретрай ставит свой маркер, а доживший первый прогон снимет его как свой
  (нет run-токена). Не чинили: цикл с таймаутами LLM физически короче TTL; run-токен —
  если всплывёт на практике.

- **[Minor]** `apps/server/src/ai/send-message.ts` (refusal-ветка) — **отказ модели персистит
  assistant-сообщение с пустым `content`**: error_card видна, но сама строка ответа пуста —

## Executor / undo

- **[Minor]** `apps/server/src/executor/undo.ts` — **undo detach бюджета может восстановить
  второго budget-parent**: internalUndo воспроизводит inverse без инвариантов (§7.8 —
  осознанно); если после detach появился новый budget-parent, undo вернёт старый аспект и
  сущность станет вторым parent'ом. Пре-существующий класс «undo без инвариантов», не
  регрессия фазы A. Кандидат: post-undo проверка инварианта с внятным отказом.
- **[Minor]** `apps/server/src/executor/executor.ts` (`prepareAttach`) — **attach
  `orbis/financial` на сущность с ≥2 parent'ами**: `assertBudgetAttachKeepsSingleParent`
  закрывает attach `orbis/budget` (ретроспективный parent), симметричный кейс — attach
  financial на сущность, уже висящую под двумя budget-parent'ами (легаси-связи), — не
  проверяется; бюджет-хук отребиндит к одному конверту, но лишние связи не снимет.
  Достижимо только при уже битых данных.
