## Retry-буфер / идемпотентность

- **[Important]** `apps/web/src/state/retry-send.ts:19` (`mapSendError`: `CONFLICT →
  'confirmed'`) + legacy-ветка `apps/web/src/features/chat/useFastPath.ts` (catch,
  `outcome === 'confirmed'`) — **клиентская конвенция «CONFLICT = подтверждённый успех»
  противоречит серверной семантике**: CONFLICT означает лишь «id занят», в т.ч. чужой
  строкой — фабрикуется успех без записи. Находка всплывала дважды (бэклог слайса 1;
  ревью B4 — там QuickAddBar починен: CONFLICT → ошибка + новый UUID, честный повтор
  владельца — replay-успех сервера). Не чинили: fast-path/retry-буфер используют
  client-generated UUIDv7, коллизия с чужой строкой практически недостижима без бага
  генератора; правильный фикс — сквозной, одним заходом с ревизией retry-буфера
  (flush-семантика, business_rejection), а не точечная правка в финале фазы.

## Грамматика §6.1

- **[Minor]** одиночная граница суммы (`txQuery.ts`: `amount>` / `amount<`) — **строгие
  >/< при плейсхолдерах «от/до»**: у грамматики §6.1 нет `>=`/`<=` — сумма ровно на
  границе фильтром «от 500» не находится. Не чинили: расширение грамматики оператором
  `>=` — отдельный заход shared+server+web (B5-1/Minor-7 финального ревью).
- **[Minor]** `\`-экранирование в кавычках (fix B5, `packages/shared/src/query/parse.ts`) —
  **тихая смена семантики `\\`-пар в старых body**: query-блоки, записанные до фикса
  с литеральными `\\` внутри кавычек (UNC-пути), теперь парсятся как экран. Узкий класс,
  миграции не делали: реальных body с таким содержимым не обнаружено.
